Contribute to DietPi

Contributions are always welcome, no matter how small or large. You can do this in many ways and this page provides everything you need to get started !

Grow in 2021

This month we are asking you to help DietPi project. DietPi is an open source product and it is free to use. We are sustained by donations and even a small one (as little as the price of a cup of coffee) would help the things move forward, covering the costs for hardware (new boards, MicroSD cards etc.), webhosting for Dietpi.com and future development.

This allows us to continue working on DietPi, make it better with every new update.



In 2020 we made 8 releases and brought 128 improvements and changes, and fixed 119 bugs. We provided new software titles, support new SBCs and brought many optimizations.
Enable us to grow in 2021 !

Why contribute to DietPi as an Open Source project ?

You can improve the software you rely on. It's empowering to be able to make changes, even small ones. Whether it’s coding, UI design, writing, or testing, if you’re looking for practice, there’s a task for you on here.

What it means to contribute ?

Create a new bug report / Report a security issue

Creating a bug report helps us improve. Providing required info, steps to replicate, and expected vs. actual behaviour you give us valuable info to understand the problem.

Create now a GitHub issue.

Security is on top of our list. Please report any security vulnerability shipped with our images or created by our scripts (in /boot/dietpi/). Please find more details in our Security Policy.

Investigate issues / Provide help

Many times the DietPi project needs help on existent items:

  • Run an investigation (e.g. an installation issue on specific software title)
  • Provide an workaround to an existent issue
  • Give help when testing is required

Browse DietPi issues on GitHub or DietPi documentation tasks on GitHub.

Give help / Answer questions

Sometimes there are open questions where you could know the answer.
Or help is needed for a specific installation (e.g. a driver installation for a specific SBC, or a check of a performance tweak etc.)

Browse DietPi issues on GitHub and find these items.

Test configurations

There is often a need to check a new installation on a specific SBC (Single Board Computer). Allocating this time to do such installation provides a big help to the overall project.


Examples:
  • A driver installation for a specific SBC
  • or check of a performance tweak.
Browse DietPi issues on GitHub and find these items.
